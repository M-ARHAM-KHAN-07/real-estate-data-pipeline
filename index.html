🔍 Key engineering insight — false positive prevention

The "Office Leases" link appears in the site nav on every page — a naive search would mark every article as matching. The fix: before any category check, decompose all nav, header, footer, and sidebar elements from the BeautifulSoup tree. Only then run the category selector chain. Results went from 100% false-positive to near-zero.

co_scraper.py — is_office_lease_article() 
def is_office_lease_article(soup: BeautifulSoup) -> bool:
    # Step 1: strip global nav to prevent false positives
    for tag in soup.find_all(["nav", "header", "footer", "aside"]):
        tag.decompose()

    # Step 2: check known article-category containers
    CATEGORY_SELECTORS = [
        "div.leases-label", "div.channel",
        "div.article-category", "ul.post-categories",
    ]
    for selector in CATEGORY_SELECTORS:
        container = soup.select_one(selector)
        if container:
            for a in container.find_all("a", href=True):
                if a["href"].endswith("/leases/office"):
                    return True

    # Step 3: fallback full-page scan (chrome-stripped)
    for a in soup.find_all("a", href=True):
        if a["href"].endswith("/leases/office"):
            return True
    return False

PerimeterX Press & Hold CAPTCHA solver Hard problem

Zillow deploys PerimeterX bot protection with a "Press & Hold" CAPTCHA — users must hold a button for 10–15 seconds to prove they're human. This is designed to defeat automated clicking. Solved using Chrome DevTools Protocol (CDP) mouse event injection.

01
Detection
Scan page source and DOM for "press & hold", "before we continue", PerimeterX element IDs. Check iframes too.
02
Human mouse path
8–15 step cubic ease-in-out mouse movement toward button center with ±2px jitter per step via CDP.
03
CDP hold
Input.dispatchMouseEvent mousePressed + real-time logging + mouseReleased after 12–16s.
04
Fallback chain
5 auto-attempts → 5-minute manual solve window → URL saved to captcha_blocked.json for retry.

# Human-like mouse path toward CAPTCHA button (cubic easing)
for step in range(steps):
    t = (step + 1) / steps
    t = t * t * (3 - 2 * t)  # smoothstep
    mx = start_x + (cx - start_x) * t + random.uniform(-2, 2)
    my = start_y + (cy - start_y) * t + random.uniform(-2, 2)
    driver.execute_cdp_cmd('Input.dispatchMouseEvent', {
        'type': 'mouseMoved', 'x': mx, 'y': my,
        'pointerType': 'mouse'
    })
    time.sleep(random.uniform(0.01, 0.04))

Anti-detection techniques

Fingerprint
undetected-chromedriver
Patches Chrome binary to remove all automation fingerprints. Removes webdriver navigator property and automation switches that PerimeterX detects.
Timing
Randomised delays
All waits use random.uniform(min, max). Page loads: 2–4s. Between listings: 3–6s. Between states: 10–20s. Bulk break every 5 listings.
User agent
UA rotation pool
Pool of 4 real Chrome UAs (macOS, Windows, Linux variants) selected randomly per driver session.
Display
Xvfb virtual display
Runs a real X11 display server on Ubuntu so Chrome operates as a true desktop browser, not a headless session. Harder for fingerprinting to detect.

Engineering notes

Challenges solved

Real problems encountered and how they were addressed

01 PerimeterX defeating normal Selenium
Standard ChromeDriver is trivially detected by PerimeterX via navigator.webdriver and automation attribute fingerprints. Switched to undetected-chromedriver which patches the binary directly. Combined with Xvfb for a real X11 environment and randomised user-agents.
02 "Press & Hold" CAPTCHA — no third-party service
This CAPTCHA can't be solved by 2Captcha or similar services — it requires a genuine sustained mouse button hold. Used CDP's Input.dispatchMouseEvent to inject mousePressed, wait 12–16 seconds with jitter, then mouseReleased. Preceded by smooth cubic-eased mouse movement from a random position to defeat motion fingerprinting.
03 Category filter false positives
Every page on the Commercial Observer site has "Office Leases" in the global navigation. Without decomposing nav/header/footer elements from the BeautifulSoup tree before checking, every article matched — 100% false positive rate. Solved by aggressive pre-filter decomposition of site chrome.
04 URL deduplication robustness (Zillow)
Zillow URLs vary between runs — trailing slashes, query params, case differences. Matching on full URL strings caused the same listings to be scraped repeatedly. Switched to ZPID extraction via regex and set-based ZPID tracking. One canonical ID per property regardless of URL variation.
